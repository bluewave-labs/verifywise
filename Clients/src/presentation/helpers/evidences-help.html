Evidence & Documents Management

The Evidence & Documents section serves as a centralized repository for all files uploaded to the VerifyWise platform. This system allows you to store, organize, and manage evidence files, compliance documents, and other supporting materials in one secure location. It's a critical component for maintaining audit trails and supporting compliance documentation.

Overview

The Evidence & Documents page provides a comprehensive file management system with the following key features:

  • Centralized File Storage: All uploaded files are stored and organized in one location
  • Project-Based Organization: Files are associated with specific projects for better organization
  • Advanced Filtering: Filter files by project to focus on specific documentation
  • Sorting Capabilities: Sort files by upload date or uploader for easy navigation
  • File Actions: Download files and access source information directly from the table

File Table Structure

The main table displays all uploaded files with the following columns:

  • File: The name of the uploaded file
  • Project Name: The project this file is associated with
  • Upload Date: When the file was uploaded to the system (sortable)
  • Uploader: The user who uploaded the file (sortable)
  • Source: The source or origin of the file (clickable with external link icon)
  • Action: Available actions for the file (download, etc.)

Project Filtering

Use the project filter dropdown to organize and view files by specific projects:

  • All Projects: View all files across all projects (default view)
  • Specific Project: Select a project to view only files associated with that project
  • No Projects Available: If no projects exist, the filter will show "No project available"

The filter dropdown is located above the file table and allows you to quickly narrow down your view to focus on specific project documentation.

Sorting Files

The table supports sorting on specific columns to help you organize your files:

  • Upload Date: Click the column header to sort by upload date (ascending/descending)
  • Uploader: Click the column header to sort by uploader name (alphabetical)

Sortable columns display arrow icons indicating the current sort direction. Click once for ascending order, click again for descending order.

File Actions

Each file in the table provides several actions through the actions menu:

  • Download: Download the file to your local machine
    • Click the actions menu (three dots) for the file
    • Select the download option
    • Confirm the download action
    • The file will be downloaded to your default download location
  • Source Access: Click on the source information to open the original source in a new tab
    • The source column shows the origin of the file
    • Clicking on the source text opens the original location in a new browser tab
    • An external link icon appears on hover to indicate it's clickable

File Information

Each file entry contains comprehensive metadata:

  • File Name: The original name of the uploaded file
  • Project Association: Which project the file belongs to
  • Upload Timestamp: Exact date and time when the file was uploaded
  • Uploader Identity: The user account that uploaded the file
  • Source Information: The original source or location where the file came from
  • File Type: The MIME type of the file (for supported file types)
  • File Size: The size of the file in bytes (when available)

Pagination and Navigation

The file table includes pagination controls for managing large numbers of files:

  • Rows Per Page: Choose how many files to display per page (5, 10, 15, 20, or 25)
  • Page Navigation: Navigate between pages using the pagination controls
  • Item Count: View the total number of files and current page range

The pagination footer shows "Showing X - Y of Z items" to help you understand your current position in the file list.

Empty State

When no files are available, the system displays a helpful empty state:

  • Visual Indicator: An empty state illustration is displayed
  • Informative Message: "There are currently no pieces of evidence or other documents uploaded."
  • Guidance: The empty state helps users understand that they need to upload files to see them in this section

File Types and Sources

The system supports various file types and sources:

  • Evidence Files: Documents that serve as evidence for compliance or audit purposes
  • Supporting Documents: Additional files that support your compliance documentation
  • External Sources: Files that originate from external systems or platforms
  • Uploaded Files: Files directly uploaded by users to the system

Best Practices

  • Organize by Project: Use the project filter to organize files by specific projects or initiatives
  • Descriptive File Names: Use clear, descriptive file names to make files easy to identify
  • Regular Review: Periodically review uploaded files to ensure they remain relevant and up-to-date
  • Source Documentation: Ensure source information is accurate to maintain proper audit trails
  • Access Management: Be mindful of who has access to sensitive evidence files
  • File Organization: Consider organizing files by upload date or uploader for easier management

Security and Compliance

The Evidence & Documents system is designed with security and compliance in mind:

  • Secure Storage: All files are stored securely within the VerifyWise platform
  • Access Control: File access is controlled by user permissions and project associations
  • Audit Trail: All file activities are logged for compliance and audit purposes
  • Data Integrity: Files maintain their original integrity and metadata

Troubleshooting

Common issues and solutions:

  • Files Not Loading: Check your internet connection and refresh the page
  • Download Issues: Ensure your browser allows downloads and check your download folder
  • Filter Not Working: Verify that projects exist and are properly configured
  • Source Links Not Opening: Check if the source URL is accessible and your browser allows pop-ups
